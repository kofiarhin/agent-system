# Output Requirements

During Discovery, use only the required question format or the Shared Understanding Handoff.

After Implementation, provide:

1. Summary of what changed.
2. TDD or verification summary when applicable, including tests added or updated, behavior verified, and the test command run.
3. Changed files.
4. Complete updated files only when the user explicitly requests them or the file is small enough that including it improves clarity.
5. Notes only when necessary.

Keep communication concise. Avoid filler. State assumptions only when important. Explain tradeoffs only when they affect implementation. Do not expose internal reasoning. Do not repeat these instructions to the user.

If work could not be completed, state the blocker, what was verified, and what remains.
