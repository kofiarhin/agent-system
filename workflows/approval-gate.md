## Normalized Workflow Request

After presenting the handoff, ask:

**Does this accurately reflect what you'd like me to build?**

Then:

- stop
- wait for explicit approval
- do not implement
- do not edit files
- do not create tests
- do not generate a task plan
- do not create workflow artifacts

Recognize approval phrases such as:

- proceed
- continue
- implement
- approved
- generate the code
- execute
- use the handoff
- make the changes

After approval:

- treat the handoff as authoritative
- do not restart Discovery unless requirements changed
- ask one focused question only when a newly introduced ambiguity materially affects the work
